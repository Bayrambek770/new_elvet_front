Nurse Dashboard Redesign - Implementation Plan
Design Reasoning & User Experience
Current Problems
Horizontal tabs take up vertical space - In a dashboard with lots of data, vertical space is precious
Tab overflow on mobile - 6 tabs don't fit well on smaller screens
No clear visual hierarchy - All tabs appear equally important
Limited scalability - Hard to add more sections without cluttering
Proposed Solution: Collapsible Sidebar Navigation
Why a Sidebar?
‚úÖ Saves vertical space - Metrics and content get full height
‚úÖ Modern UX pattern - Users expect sidebars in admin/dashboard apps (like Gmail, Slack, etc.)
‚úÖ Better mobile experience - Can fully hide on small screens, giving content 100% width
‚úÖ Scalable - Easy to add/remove navigation items
‚úÖ Clear hierarchy - "Main" is the home/default view
Auto-hide Behavior
The sidebar will:

Start collapsed on page load (showing only icons)
Expand on hover (desktop) or toggle button (all devices)
Remember state using localStorage
Auto-collapse on mobile after selection to maximize content space

Implementation Notes
Use query parameters (?view=) instead of routes to keep single-page experience
Main view shows the dashboard metrics (Daily rate, Tasks, Completed today, Medicines in stock)
Other views replace the main content area while keeping header and sidebar

State Management
const [sidebarOpen, setSidebarOpen] = useState(() => {
  return localStorage.getItem('nurseSidebarOpen') === 'true';
});
const [activeView, setActiveView] = useState<'main' | 'medical-cards' | 'second-visit' | 'medicines'>(() => {
  const params = new URLSearchParams(location.search);
  return params.get('view') || 'main';
});
